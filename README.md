# ドローン在庫管理 API (DroneInventorySystem)

システム
製造業クライアント向けのドローン在庫管理システム です。

## 📁 ドキュメント構成

### 設計ドキュメント

設計フェーズで作成されたドキュメント群は、[docs/設計/](docs/設計/) フォルダーに配置されています。

#### 📂 共通設計ドキュメント

[docs/設計/共通/](docs/設計/共通/) フォルダーには、システム全体の共通設計ドキュメントが配置されています。

##### 📄 [API システム概要.md](docs/設計/共通/APIシステム概要.md)

- **目的**: システム全体のアーキテクチャと設計方針を定義
- **対象読者**: システムアーキテクト、開発チーム、インフラエンジニア
- **主要内容**:
  - **API 中心設計**: REST API サーバーとしてのアーキテクチャパターン
  - **システム構成**: 全体構成と外部システム連携
  - **業務プロセス**: 部品・製品の入荷・出荷プロセス詳細
  - **技術スタック**: バックエンド技術選定と構成
  - **ビジネスルール**: データ整合性とトランザクション原則

##### 📄 [API 仕様(共通).md](<docs/設計/共通/API仕様(共通).md>)

- **目的**: 全 API 共通の仕様とガイドラインを定義
- **対象読者**: システムアーキテクト、バックエンド開発チーム、外部システム連携担当
- **主要内容**:
  - **認証・認可**: JWT 認証とシステム権限制御
  - **API 一覧**: 部品・製品在庫管理 API の全体概要
  - **共通仕様**: レスポンス形式、エラーハンドリング、冪等性制御
  - **セキュリティ**: 通信セキュリティと制限事項

##### 📄 [データ要件.md](docs/設計/共通/データ要件.md)

- **目的**: データベース設計とデータ構造を定義
- **対象読者**: データベースアーキテクト、バックエンド開発チーム、システムアーキテクト
- **主要内容**:
  - **データ設計方針**: 業務分離、プロセス対応、トレーサビリティ
  - **ER 図**: システム全体の概念データモデル
  - **テーブル定義**: 各テーブルの詳細仕様（MySQL 8.0 対応）
  - **命名規則**: データベースオブジェクトの命名規則

[docs/設計/製品在庫管理/](docs/設計/製品在庫管理/) フォルダーは、製品在庫管理に関する API 仕様が配置されています。

**📍 開発開始前の必読資料**:
設計ドキュメントは開発開始前に必ず確認し、システムの技術仕様と設計方針を理解してください。共通設計ドキュメントから読み始めることを推奨します。
